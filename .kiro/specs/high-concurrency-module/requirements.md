# 需求文档：高并发电商系统模块

## 概述

基于现有 Spring Cloud 微服务电商项目，新增高并发基础设施模块，以最小侵入方式为订单业务引入多级缓存、消息队列异步削峰、接口限流、数据库读写分离等能力。

## 用户故事

### US-1：多级缓存（Caffeine + Redis）
作为系统运维人员，我希望订单查询走多级缓存（L1 本地缓存 + L2 Redis），以减少数据库压力并提升查询响应速度。

**验收标准：**
- AC-1.1：订单详情查询优先从 Caffeine 本地缓存获取，未命中则查 Redis，再未命中才查数据库
- AC-1.2：数据库回源后自动回填 L1 和 L2 缓存
- AC-1.3：缓存穿透防护：不存在的 key 写入空值占位符（60s TTL）
- AC-1.4：缓存击穿防护：热点 key 过期时通过分布式锁保证仅一个线程回源
- AC-1.5：缓存雪崩防护：TTL 加随机偏移量
- AC-1.6：订单写入后主动清除相关缓存

### US-2：RabbitMQ 消息队列异步削峰
作为系统架构师，我希望订单创建通过 RabbitMQ 异步处理，将瞬时高流量转为平稳消费。

**验收标准：**
- AC-2.1：订单创建请求发送到 RabbitMQ，快速返回订单号
- AC-2.2：消费者从队列拉取消息并写入数据库
- AC-2.3：消费端基于 messageId 实现幂等，重复消息仅处理一次
- AC-2.4：支持延迟消息（订单超时取消），基于 DLX + TTL 实现
- AC-2.5：消费失败通过 basicNack 重试，超过 3 次进入死信队列

### US-3：Sentinel 接口限流
作为系统运维人员，我希望对关键接口进行 QPS 限流，防止系统被突发流量打垮。

**验收标准：**
- AC-3.1：Gateway 层全局限流过滤器，超限返回 429
- AC-3.2：Service 层通过 @SentinelResource 注解实现接口级限流
- AC-3.3：限流触发时返回友好的降级响应

### US-4：数据库读写分离
作为数据库管理员，我希望读请求路由到从库、写请求路由到主库，以提升数据库吞吐量。

**验收标准：**
- AC-4.1：通过 @ReadOnly 注解标记只读方法，自动路由到从库
- AC-4.2：未标记的方法默认路由到主库
- AC-4.3：基于 AOP + ThreadLocal 实现动态数据源切换
- AC-4.4：请求结束后 ThreadLocal 自动清理

### US-5：连接池与配置优化
作为开发人员，我希望 HikariCP 连接池和 RabbitMQ 连接参数合理配置。

**验收标准：**
- AC-5.1：HikariCP 配置 maximum-pool-size=20、leak-detection-threshold=60000
- AC-5.2：RabbitMQ 配置 publisher-confirms、consumer prefetch
- AC-5.3：Redis 连接池配置合理参数

## 正确性属性

1. **缓存一致性**：∀ key, 数据修改后缓存在 TTL 内失效或被主动清除
2. **消息幂等性**：∀ messageId, 同一消息消费 N 次仅产生一条订单记录
3. **限流有效性**：∀ resource, 1 秒窗口内通过请求数 ≤ QPS 阈值
4. **读写分离正确性**：@ReadOnly 方法路由到 Slave，写操作路由到 Master
5. **库存一致性**：订单创建时 Redis 库存扣减与数据库写入保持最终一致
